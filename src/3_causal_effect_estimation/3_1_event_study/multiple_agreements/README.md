# WindowGenerator Documentation

## Overview

The `WindowGenerator` class creates balanced panel datasets with treatment and control windows for event study analysis using Callaway and Sant'Anna's Difference-in-Differences (CSDID) methodology.

## Usage

This tool is implemented in two notebooks:
- [1_country_panel_window_generator.ipynb](1_country_panel_window_generator.ipynb) - Creates windows at the country level
- [1_conflict_panel_window_generator.ipynb](1_conflict_panel_window_generator.ipynb) - Creates windows at the conflict level

The generated windows are then used as input for event study regressions in Stata do files.

---

## Data Preparation

### Required Columns

Before using `WindowGenerator`, your panel data must have:

1. **Unit identifier**: `isocode` (country panel) or `conflict_id` (conflict panel)
2. **Time identifier**: `year_mo` (datetime format)
3. **Treatment indicator**: Binary column (0/1) indicating treatment status
4. **Outcome variable**: e.g., `log_best` (log of battle-related deaths)

### Create Time Index Column

**CRITICAL STEP**: You must create a consecutive integer time index for each unit starting from 0, this step is already included in the notebooks, but here’s how it works:

```python
# For country panel
df['year_mo'] = pd.to_datetime(df['year_mo'])
df['isocode_t'] = df.groupby('isocode').year_mo.transform(lambda x: pd.factorize(x, sort=True)[0])
df = df.sort_values(['isocode', 'year_mo'])
```

```python
# For conflict panel
df['year_mo'] = pd.to_datetime(df['year_mo'])
df['conflict_id_t'] = df.groupby('conflict_id').year_mo.transform(lambda x: pd.factorize(x, sort=True)[0])
df = df.sort_values(['conflict_id', 'year_mo'])
```

**What this does**:
- `pd.factorize(x, sort=True)[0]` assigns consecutive integers (0, 1, 2, ...) to each unique date within each unit
- This ensures time periods are numbered sequentially without gaps, which is required by WindowGenerator
- The transformation is done **within each unit** (via `groupby`), so each unit has its own time sequence starting from 0

## WindowGenerator Parameters

### Initialization

```python
from window_generator import WindowGenerator

wg = WindowGenerator(
    dataframe=df,                          # Panel dataset (pd.DataFrame)
    frame_size=[18, 18],                   # [pre-periods, post-periods] around event
    unit_column='isocode',                 # Unit identifier column name
    time_column='isocode_t',               # Consecutive time index column name
    treatment_column='ceasfire_agreements_mentions'  # Binary treatment indicator
)
```

**Parameter Details**:
- `dataframe`: Must be balanced (same time points for each unit, no gaps)
- `frame_size`: List of 2 integers `[pre, post]` defining window size around treatment
  - Example: `[18, 18]` = 17 months before + treatment month + 18 months after (36 months total)
- `unit_column`: Column identifying units (countries or conflicts)
- `time_column`: Consecutive integer time index (0, 1, 2, ...) created in data preparation step
- `treatment_column`: Binary (0/1) column with no nulls

### Generate Treatment Windows

```python
buffer_size_treated = [18, 1]  # [pre-buffer, post-buffer]
wg.generate_treatment_windows(buffer_size=buffer_size_treated)
```

**Buffer Size**: Defines periods around treatment where no other treatments should occur
- Example: `[18, 1]` = require 18 months before and 1 month after with no other treatments
- Ensures clean treatment windows without contamination

### Generate Control Windows

Controol windows can be generated by random matching or KNN matching based on covariates. The results obtained up to now are based on random matching.

```python
# Option 1: Random Matching
wg.generate_control_windows(
    buffer_size=[18, 18],        # [pre-buffer, post-buffer] for controls
    k=5,                          # Number of control windows per treated window
    t=1,                          # Max times a control can be reused
    matching_method='random'
)
```

```python
# Option 2: KNN Matching
matching_params = {
    'matching_columns': ['log_gdp_pc_current_usd_lag1'], #can be multiple columns
    'distance_threshold': 0.2     # Normalized Euclidean distance threshold
}

wg.generate_control_windows(
    buffer_size=[18, 18],
    k=5,
    t=1,
    matching_method='knn',
    matching_params=matching_params
)
```

**Parameters**:
- `buffer_size`: Periods where controls should have no treatments
- `k`: Number of control windows to match per treated window
- `t`: Maximum reuse of each control window (1 = without replacement)
- `matching_method`: `'random'` or `'knn'`
- `matching_params` (for KNN only):
  - `matching_columns`: Features for matching (will be normalized)
  - `distance_threshold`: Maximum normalized distance to consider a match

### Access Results
In this step we combine the treated and control windows into a single dataset.

```python
# Combined dataset with treated and control windows
df_windows = wg.combined_windows

# Separate datasets
df_treated = wg.treated_windows
df_control = wg.control_windows
```

**New columns added**:
- `window_id`: Unique identifier for each window (format: `{unit}_{time}`)
- `window_t`: Time relative to treatment (-18, -17, ..., 0, ..., 17, 18)
- `is_treated_window`: Binary indicator (1=treated, 0=control)
- `matched_treated_window_id`: For control windows, which treated window they match
